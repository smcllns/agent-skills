## Mission

Switch the trigger syntax from `#`-prefix to `@`-prefix, and add support for caller-overridable trigger sets. One focused round.

## Why the switch

The `#claude` / `#codex` / `#agent` triggers collide with Obsidian's native `#tag` system. Many users already have `#claude` / `#codex` tags in their notes as topic labels — the plugin currently hijacks all of them, which is rude. `@` reads more naturally as addressing ("@claude do X"), doesn't collide with hashtag conventions, and matches the in-callout speaker convention (`@claude:`, `@sam:`).

1. **SKILL.md frontmatter description** — replace any `#`-mention or "hashtag" phrasing with `@`-mention phrasing. The user has already lightly edited this line; don't undo their edit, just adjust references to `#` → `@`.
2. **Regex in `## Scanning for unresolved asks`** — change `#(agent|claude|codex)` to `@(agent|claude|codex)`. Word-boundary logic stays the same. Reconsider the `^[^>]` blockquote-skip clause — still correct.
3. **`## Ask shapes` table** — row 1's `#agent` becomes `@agent`.
4. **All examples** throughout SKILL.md — `#claude foo` → `@claude foo`.

## Hand-off questions you might face

- **"What about backward compatibility with old `#claude` asks already in vaults?"** Hard switch, not dual-support. The regex changes from `#` to `@`. Existing `#claude` callouts in vaults still render fine because the directive text inside a `[!DONE]-` is preserved-but-skipped (the `^[^>]` clause). Only **unresolved** `#claude` lines stop being picked up — those need migration.
- **"Should we also support trigger names beyond claude/codex/agent in the default set?"** No. Defaults stay `@claude`, `@codex`, `@agent`. Customization happens through the override mechanism.
- **"Should I add a configuration UI / wizard?"** No. The skill is stateless. The override mechanism is "pass a different trigger list in the prompt."
